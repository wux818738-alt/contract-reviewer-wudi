## 配置结构

```json
{
  "$schema": "contract-type-config",
  "contract_type": "合同类型名称",
  "contract_type_en": "英文代码",
  "description": "描述",
  "version": "版本号",
  "applicable_laws": ["适用法律"],
  "keywords_for_detection": ["检测关键词"],
  "key_clauses": [...],
  "common_risks": [...],
  "position_guidance": {...}
}
```

## 字段说明

### key_clauses（关键条款）

每个条款包含：
- `name`: 条款名称
- `required`: 是否必需
- `keywords`: 识别关键词
- `check_points`: 检查要点列表
- `suggestion`: 通用建议

### check_points（检查要点）

- `item`: 检查项名称
- `description`: 描述
- `risk_level`: 风险等级（致命/重大/一般/轻微）
- `regex_pattern`: 可选的正则匹配模式

### common_risks（常见风险）

- `id`: 风险编号
- `name`: 风险名称
- `description`: 风险描述
- `severity`: 严重程度（高风险/中风险/低风险）
- `pattern`: 匹配模式
- `negative_pattern`: 是否为否定匹配（true=未匹配时触发）
- `suggestion`: 修改建议
- `legal_basis`: 法律依据

### position_guidance（立场指导）

- `priority_checks`: 优先检查项
- `key_concerns`: 关键关注点
- `avoid`: 应避免的问题

## 使用方式

1. **自动检测合同类型**：根据 `keywords_for_detection` 匹配
2. **加载审核要点**：根据检测到的类型加载对应 JSON
3. **执行针对性检查**：按 `key_clauses` 和 `common_risks` 检查
4. **生成立场建议**：根据 `position_guidance` 生成建议

## 扩展新类型

1. 复制 `template.json`
2. 修改合同类型信息
3. 填写关键条款和检查要点
4. 定义常见风险和立场指导
5. 保存为 `{type-code}.json`
6. 更新本 README
